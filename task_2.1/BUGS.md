# Баг-репорты

Баги подтверждены прогоном автотестов на сервере `https://qa-internship.avito.com` от 05.04.2026.



## BUG-001: Ответ POST /api/1/item не соответствует Postman-контракту

Серьёзность: Major

Описание: Postman-коллекция описывает ответ POST как объект с полями `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`. Реальный API возвращает `{"status": "Сохранили объявление - <uuid>"}`. ID объявления приходится извлекать из строки.

Шаги воспроизведения:
1. POST `https://qa-internship.avito.com/api/1/item` с валидным телом.

Фактический результат: `{"status": "Сохранили объявление - 458f62e2-143e-4a91-8e6c-cc72ce0ec4db"}`

Ожидаемый результат (по Postman): `{"id": "...", "sellerId": ..., "name": "...", "price": ..., "statistics": {...}, "createdAt": "..."}`

Окружение: API `https://qa-internship.avito.com`, POST `/api/1/item`

## BUG-002: Создание объявления с отрицательной ценой возвращает 200

Серьёзность: Critical

Описание: API принимает отрицательные значения `price` и создаёт объявление. Цена не может быть отрицательной.

Шаги воспроизведения:
1. POST с телом `{"name": "Test", "price": -100, "sellerId": 111111, "statistics": {"contacts": 1, "likes": 0, "viewCount": 0}}`

Фактический результат: Статус 200. `{"status": "Сохранили объявление - 2e154191-..."}`

Ожидаемый результат: Статус 400. Ошибка валидации: цена не может быть отрицательной.

Окружение: API `https://qa-internship.avito.com`, POST `/api/1/item`

---

## BUG-003: Создание объявления с sellerId = 0 возвращает 200

Серьёзность: Major

Описание: API принимает `sellerId = 0` и создаёт объявление. Идентификатор продавца должен быть положительным.

Шаги воспроизведения:
1. POST с `"sellerId": 0`

Фактический результат: Статус 200. `{"status": "Сохранили объявление - 9cffb1e6-..."}`

Ожидаемый результат: Статус 400. Ошибка валидации.

Окружение: API `https://qa-internship.avito.com`, POST `/api/1/item`



## BUG-004: Создание объявления с отрицательным sellerId возвращает 200

Серьёзность: Major

Описание: API принимает отрицательные значения `sellerId` (например, -1) и создаёт объявление. В базе уже накоплены сотни записей с `sellerId = -1` от разных тестировщиков.

Шаги воспроизведения:
1. POST с `"sellerId": -1`

Фактический результат: Статус 200. Объявление создано.

Ожидаемый результат: Статус 400. `sellerId` должен быть положительным целым числом.

Окружение: API `https://qa-internship.avito.com`, POST `/api/1/item`



## BUG-005: GET /api/1/-1/item возвращает 200 с объявлениями вместо 400

Серьёзность: Major

Описание: Запрос объявлений по отрицательному `sellerId` возвращает массив объявлений (сотни записей) вместо ошибки 400. Это следствие BUG-004 — API позволяет создавать объявления с `sellerId = -1`.

Шаги воспроизведения:
1. GET `https://qa-internship.avito.com/api/1/-1/item`

Фактический результат: Статус 200. Массив из сотен объявлений.

Ожидаемый результат: Статус 400. Ошибка: отрицательный `sellerId` невалиден.

Окружение: API `https://qa-internship.avito.com`, GET `/api/1/-1/item`



## BUG-006: Создание объявления с price = 0 возвращает 400

Серьёзность: Minor (спорный, зависит от бизнес-требований)

Описание: API отклоняет `price = 0` с кодом 400. При этом отрицательная цена (BUG-002) принимается. Если бесплатные объявления допустимы — это баг. Если нет — поведение корректно, но непоследовательно с обработкой отрицательных цен.

Шаги воспроизведения:
1. POST с `"price": 0`

Фактический результат: Статус 400.

Ожидаемый результат: Статус 200 (если бесплатные допустимы) или 400 с консистентным поведением для отрицательных цен.

Окружение: API `https://qa-internship.avito.com`, POST `/api/1/item`



## BUG-007: Несогласованность имени поля sellerID / sellerId в Postman-контракте

Серьёзность: Minor

Описание: В Postman-коллекции тело запроса POST использует `sellerID` (с заглавными ID), а тело ответа — `sellerId` (camelCase). Реальный API принимает `sellerId`.

Окружение: Postman-коллекция `qa-internship Copy 2`
